---
title: "Users"
description: "Manage users (profiles) and their membership roles within your organization."
---

Users in Probo are represented as **profiles**. A profile holds a person's identity information and is linked to an organization through a membership with a specific role.

## The Profile object

<ResponseField name="id" type="string" required>
  Global ID (GID) of the profile.
</ResponseField>

<ResponseField name="organization_id" type="string" required>
  GID of the organization this profile belongs to.
</ResponseField>

<ResponseField name="full_name" type="string" required>
  Full name of the user.
</ResponseField>

<ResponseField name="email_address" type="string" required>
  Primary email address.
</ResponseField>

<ResponseField name="additional_email_addresses" type="string[]" required>
  Additional email addresses associated with the user.
</ResponseField>

<ResponseField name="kind" type="string | null" required>
  User kind or employment type (e.g. employee, contractor).
</ResponseField>

<ResponseField name="position" type="string | null">
  Job title or position.
</ResponseField>

<ResponseField name="contract_start_date" type="string | null (date-time)">
  Contract or employment start date.
</ResponseField>

<ResponseField name="contract_end_date" type="string | null (date-time)">
  Contract or employment end date.
</ResponseField>

<ResponseField name="created_at" type="string (date-time)" required>
  ISO 8601 timestamp of when the profile was created.
</ResponseField>

<ResponseField name="updated_at" type="string (date-time)" required>
  ISO 8601 timestamp of the last update.
</ResponseField>

## MembershipRole enum

Controls what actions a user can perform within an organization.

| Value | Description |
|-------|-------------|
| `OWNER` | Full administrative access, including billing and organization deletion |
| `ADMIN` | Can manage users, settings, and all resources |
| `EMPLOYEE` | Standard access to compliance resources |
| `VIEWER` | Read-only access |
| `AUDITOR` | Read-only access intended for external auditors |

## The Membership object

<ResponseField name="id" type="string" required>
  Global ID (GID) of the membership.
</ResponseField>

<ResponseField name="role" type="MembershipRole" required>
  The user's role in the organization.
</ResponseField>

<ResponseField name="created_at" type="string (date-time)" required>
  ISO 8601 timestamp of when the membership was created.
</ResponseField>

## Operations

### listUsers

Returns all users in an organization.

**MCP tool:** `listUsers`

#### Parameters

<ParamField body="organization_id" type="string" required>
  GID of the organization.
</ParamField>

<ParamField body="size" type="integer">
  Number of results per page. Defaults to 20.
</ParamField>

<ParamField body="cursor" type="string">
  Pagination cursor from a previous response's `next_cursor`.
</ParamField>

<ParamField body="order_by" type="object">
  <Expandable title="order_by fields">
    <ParamField body="field" type="string">
      Sort field. One of: `CREATED_AT`, `FULL_NAME`, `KIND`.
    </ParamField>
    <ParamField body="direction" type="string">
      Sort direction. One of: `ASC`, `DESC`.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="filter" type="object">
  <Expandable title="filter fields">
    <ParamField body="exclude_contract_ended" type="boolean">
      When `true`, excludes users whose contract end date is in the past.
    </ParamField>
  </Expandable>
</ParamField>

#### Response

<ResponseField name="users" type="Profile[]" required>
  Array of profile objects.
</ResponseField>

<ResponseField name="next_cursor" type="string">
  Cursor for the next page. Absent when there are no more results.
</ResponseField>
