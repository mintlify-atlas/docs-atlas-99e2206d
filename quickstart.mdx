  <Tab title="Cloud-hosted">
    The fastest way to get started is with the cloud-hosted version of Probo.

    <Steps>
      <Step title="Create your account">
        Visit [app.getprobo.com](https://app.getprobo.com) and sign up for a free account.
      </Step>
      <Step title="Create your organization">
        After signing in, create an organization for your company. This is the top-level container for all your compliance data.
      </Step>
      <Step title="Select a compliance framework">
        Navigate to **Frameworks** and select the framework you want to pursue — SOC2, ISO 27001, GDPR, HIPAA, or others. Probo will populate a pre-built set of controls to work through.
      </Step>
      <Step title="Invite your team">
        Go to **Settings → Team** and invite colleagues. Assign roles (Owner, Admin, Employee, Viewer, Auditor) based on their responsibilities.
      </Step>
    </Steps>

  <Tab title="Self-hosted">
    Run Probo on your own infrastructure using Docker Compose and a single Go binary.

    ## Prerequisites

    Before you begin, make sure you have:

    - **Go 1.21+** — [download](https://go.dev/dl/)
    - **Node.js 22+** — [download](https://nodejs.org/)
    - **Docker** — [download](https://docs.docker.com/get-docker/)
    - **mkcert** — for local TLS (`brew install mkcert` or [see docs](https://github.com/FiloSottile/mkcert))

    ## Setup

    <Steps>
      <Step title="Clone the repository">
        ```bash
        git clone --recurse-submodules https://github.com/getprobo/probo.git
        cd probo
        ```

        <Note>
          The `--recurse-submodules` flag is required — Probo uses Git submodules for some dependencies.
        </Note>
      </Step>
      <Step title="Install dependencies">
        ```bash
        # Install Go dependencies
        go mod download

        # Install Node.js dependencies
        npm ci
        ```
      </Step>
      <Step title="Start infrastructure services">
        ```bash
        make stack-up
        ```

        This starts PostgreSQL, SeaweedFS (S3-compatible file storage), Mailpit (local email), Chrome (for PDF generation), and the observability stack via Docker Compose.
      </Step>
      <Step title="Build the project">
        ```bash
        make build
        ```

        This compiles the Go binary and bundles the React frontend into `bin/probod`.

        <Tip>
          For faster iteration on backend changes only, skip the frontend build: `SKIP_APPS=1 make build`
        </Tip>
      </Step>
      <Step title="Start Probo">
        ```bash
        bin/probod -cfg-file cfg/dev.yaml
        ```

        The application will be available at **http://localhost:8080**.
      </Step>
    </Steps>

    <Warning>
      The `cfg/dev.yaml` configuration uses insecure placeholder secrets and is intended for local development only. Never use it in production. See [Deployment](/deployment) for production configuration.
    </Warning>

    ## Testing custom domains (optional)

    To test the trust center custom domain feature locally, add the CNAME target to your hosts file:

    ```bash
    # macOS / Linux: edit /etc/hosts
    # Windows: edit C:\Windows\System32\drivers\etc\hosts
    127.0.0.1 custom.getprobo.com
    ```

    The CNAME target is configured in `cfg/dev.yaml` under `custom-domains.cname-target`.
